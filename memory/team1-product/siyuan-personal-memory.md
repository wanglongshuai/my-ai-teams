### IM 系统代码分析（2026-04）
完整分析了现有 IM 系统代码结构，产出能力清单并上报志远。

**已有功能核心**：文本消息+Markdown渲染、SSE实时推送、消息历史(100条)、打字状态；预设群+单聊+自定义群；任务系统（并行执行、3层嵌套、任务看板）；成员信息面板+组织架构视图。

**明显缺失（已验证）**：
- 高优先：@人功能（代码无解析逻辑）、消息引用/回复（无reply_to字段）
- 中优先：消息撤回、未读消息计数（CSS有`.unread-badge`但逻辑未实现）、表情回应、文件/图片发送
- 低优先：消息搜索、群公告/置顶

**结论**：系统是可运行MVP，核心聊天+任务协作已通，IM基础体验存在明显短板。

## 已完成的需求验证任务

### 饮食拦截 App · iOS MVP（2026-04-08）
- 原始痛点：真实（行为经济学支撑，评分 7/10）
- 产品形态（拦截+倒计时）：高风险伪假设（评分 3/10）
- 综合需求真实性评分：4/10
- 核心发现：用户在「脆弱时刻」会绕过任何限制型工具（断点D），这是无法用产品设计解决的根本性问题
- 典型「用户说A但其实需要B」案例：用户说要「拦截工具」，实际需要的是「建立健康习惯的正向激励系统」
- 建议形态调整：正向激励型（「忍住了」记录+可视化）或认知干预型（弹出反思提示），绕开 Screen Time API 硬限制
- 放行前置条件（3项）：技术 PoC 通过 + 落地页 15% 付款触达率 + 10 访谈中 6 人有强烈干预需求
