## 已验证的市场洞察

> 记录经过用户研究验证的市场痛点和机会

- **饮食拦截 App 变现模式**（2026-04-08，明远）：订阅制在该产品上 CAC > LTV，不成立。推荐买断制 28-48 元，参考 Forest/Streaks 定价逻辑。7 日留存 20% 导致 30 日留存极低，LTV 约等于一个月费，低于 iOS 渠道 CAC（50-80 元）。
- **该赛道收入天花板**（2026-04-08，明远）：SOM 估算不超过 50 万有效用户，20% 付费率下天花板约 1900 万元，适合小团队精品运营，不适合融资驱动高增长。12 个月 ARR 保守估算约 15-20 万元（MVP 阶段）。
- **竞品矩阵与缺口**（2026-04-08，晓敏）：One Sec / Opal / Forest / Streaks 四款主要竞品均无针对"饮食/外卖 App"的场景化拦截，场景专一是真实的差异化空白。One Sec 是形态最近的竞品（触发延迟干预），但其摩擦仅 1 秒，与 10 分钟倒计时有本质区别。整个赛道资本热度低，无新一轮融资信号，符合精品产品定位，不支持融资驱动路径。

## 识别出的伪需求

> 记录被验证为伪需求的方向，避免重复探索

- **「拦截+倒计时」干预形态**（2026-04-08，思远）：以强制阻断为核心的产品范式是伪需求形态。原始痛点（疲惫冲动点餐）真实，但用户在真实冲动时刻会绕过任何摩擦工具（卸载/关闭权限），行为科学支持奖励型干预优于限制型。10 分钟倒计时无实验依据，存在逆反效应风险（Reactance Theory）。Screen Time API 技术硬限制进一步封死该形态。综合评分 4/10，不推荐以此形态直接进入开发，需先通过三项前置验证（技术 PoC + 落地页付费测试 + 10 人脆弱时刻访谈）。

## 有效的研究方法

> 记录在用户研究中有效的方法和技巧

- **「脆弱时刻」情境还原访谈**（2026-04-08，建国）：针对自律类产品，访谈核心不是问功能偏好，而是还原「上一次你失控时，从开始到最终决定，你的内心独白是什么」。用户在清醒时对自己冲动状态的描述往往高度失真，要引导他们逐步还原具体场景细节，才能接近真实行为。
- **安装触发点追溯访谈**（2026-04-08，建国）：自律工具的安装几乎都发生在某个负面事件（体检报告、照镜子时某个念头、某段对话）之后的「情绪窗口期」内。找准这个触发点，是精准获客和文案设计的关键前提。

## 教训和踩坑记录

> 记录失败案例和经验教训

- **iOS Screen Time API 的三个硬性坑**（2026-04-08，慧敏）：FamilyControls 无法在拦截流程中展示自定义 UI；ManagedSettings 封锁不支持"10分钟后自动解锁"的动态行为；entitlement 申请需提前 1-4 周，且有被拒风险。凡是涉及此 API 的产品方向，必须先做技术 PoC。
- **饮食拦截 App 风险评估结论**（2026-04-08，慧敏）：综合风险等级高，推荐谨慎推进。核心决策节点是"2 周技术 PoC"，验证 Screen Time API 能否支撑倒计时拦截 UI；PoC 结论出来前不投入其他开发资源。停止条件：PoC 无解且无替代方案，或 entitlement 被拒且拒绝理由涉及场景不符，或 7 日留存低于 10%。
- **Screen Time 类产品的 entitlement 申请原则**（2026-04-08，慧敏）：必须在开发第一天并行提交申请，不等 PoC 完成；申请材料强调"个人自我管理"而非"拦截/封锁"；参考 Opal、One Sec 已过审的描述策略。
- **自律工具类产品失败路径模式**（2026-04-08，慧敏）：最常见失败路径是技术降级导致产品差异化消失（强干预降为弱提醒），与系统原生功能无明显区别，用户付费意愿丧失。早期预警：TestFlight 拦截成功率 < 30% 且持续下降。
- **10 分钟倒计时的竞品对照风险**（2026-04-08，晓敏）：One Sec 成功核心是"极低摩擦（1秒）+用户自主感"，10 分钟是其 600 倍摩擦，Reactance Theory 预测用户逆反率高。该时长参数必须通过落地页 A/B 测试或访谈验证，不可在 PoC 前锁定。

---

- **通知引导方案的本质判断**（2026-04-08，慧敏）：需求文档推荐的「通知引导」路径绕过了 FamilyControls 无法展示自定义 UI 的技术限制，但将产品核心价值从「强制摩擦干预行为」降级为「提醒摩擦干预意识」，两者不是同一类产品。用户在冲动时刻忽略通知的成本几乎为零，保守估计冲动中止率从原始假设 40% 降至 15-20%，用户付费意愿随之大幅削弱。
- **通知点击率是通知引导方案的核心判断指标**（2026-04-08，慧敏）：TestFlight 阶段通知点击率必须 > 30%，否则拦截形同虚设，需在 PoC 结论出来后重新定义产品形态而非继续 MVP 开发。
- **后台计时解锁可靠性风险**（2026-04-08，慧敏）：10 分钟倒计时若 App 被 iOS 挂起，计时暂停和解锁通知未触发是可预见失败场景，需通过 Background Task 方案解决，PoC 阶段必须验证解锁成功率 > 95%。
- **DeviceActivityMonitor 触发延迟风险**（2026-04-08，慧敏）：监控触发基于时间段活动阈值而非实时事件回调，实测延迟可能达数秒至数十秒，需 PoC 实机验证：连续 10 次触发成功率 > 80%，延迟 < 5 秒。

*最后更新：2026-04-08，慧敏完成饮食拦截 App 通知引导技术方案可行性评估，核心结论：技术上可跑通但产品有效性存疑，PoC 阶段必须测量通知点击率，低于 30% 需重新定义产品形态；建议支持推进 PoC，不支持直接进入 MVP 开发*
