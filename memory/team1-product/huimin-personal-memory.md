## 积累的风险评估方法论

- **技术风险优先原则**：当产品核心交互依赖特定 API 时，技术可行性验证的优先级高于一切，PoC 结论出来前不投入其他资源
- **停止条件必须在决策前制定**：事后制定停止条件会受沉没成本影响，必须在资源投入前写清楚
- **entitlement 类审核并行启动原则**：凡需第三方审批的事项，第一天即启动申请，不等开发完成
- **失败路径模式识别**：强干预产品最常见失败路径是技术降级导致差异化消失，需在 PoC 阶段即设计替代方案
